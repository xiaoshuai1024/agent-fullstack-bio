# 用户交互偏好（MUST）

本文档记录用户与 Agent 交互的长期偏好，Agent 必须遵守。

---

## 语言（全局）

- **默认中文**：向用户发出的**提问、确认、选项**（含 `AskQuestion` 弹窗的标题、题干、选项）**默认使用中文**
- **简体中文叙述（用户指定）**：当用户明确要求助手以**简体中文**回复，且**叙述性正文**不使用英文或繁体中文时，须遵守；代码片段、命令、路径、接口名、错误码等标识可保留原文
- **禁止用英文询问用户**，除非用户在本对话中明确要求使用英文
- 代码片段、命令、接口路径、云效工作项 ID、错误码等标识可保留原文

---

## 知识库语义

- 当用户提到"知识库""保存到知识库""写入知识库"时，默认指 **MCP 知识库（memory）**，不是 `docs/` 下文档库
- 仅允许使用 MCP server 名称 `memory`，禁止使用 `user-claude-mem` 或任何 `claude-mem` 相关别名
- 若再出现"将知识库误解为文档库"的偏差，必须先按 `memory` MCP 工具链执行检索/写入

---

## 交互方式

### 头脑风暴
- 用户明确要求「头脑风暴使用弹窗选择交互」时，后续 brainstorming 阶段优先使用可选项弹窗（多选/单选）
- 仅在用户主动改回聊天式交互、或弹窗无法表达该问题时，才改用文本提问

### 仓库脚本与实现（本仓库）
- **未经用户明确同意，不得擅自修改脚本实现**（如 `scripts/feishu/*.sh` 等）：若需改动，须先说明目的与范围并征得用户同意后再改

### 开发执行
- 用户要求「尽量并行并持续执行、开发阶段不打断」时：
  - 计划阶段一次性完成关键决策收集（通过弹窗为主）
  - 进入开发执行后，默认持续推进并行任务，不因常规实现细节中断
  - 仅在无法自行解决的阻塞问题（权限、外部依赖不可用、风险性不可逆操作）时，才允许中断并询问用户

### 提交代码
- 用户明确要求「提交代码」「提交修改」或等价语义时，按完整提交流程：`commit + push`（非仅 commit）
- 用户请求创建 PR/MR（或等价语义）时，默认触发 PR 编排工具（`scripts/codeup/create-pr-all.sh`）：先 `--dry-run` 打印计划并征得用户确认

---

## 云效相关

### 工作项状态（MUST）
- **开始/完成/工时**：根据**用户要求**与**实质进展**推断 → **必须 AskQuestion 弹窗（中文）** → 用户同意后再写入
- **禁止**擅自写入工作项状态、起止时间、工时
- **Token 已配置时**：拉取/缓存/MR 等默认直接执行；工作项状态仍须询问

### PR/MR 创建（MUST）
- 当用户要求创建 PR/MR 时：
  - 必须先询问是否需要协助检查环境（Cursor Agent 登录、MCP 配置、YUNXIAO token、组织权限）
  - 优先使用云效 MCP Server 创建
  - 禁止直接使用 `gh pr create` 或 GitHub CLI
  - 若 `scripts/codeup/.env` 缺失，必须先询问用户是否自动创建该文件

### 首次 MCP 初始化（无 token）
- 必须提供新手引导：token 创建 URL、MCP 初始化文档入口、分步操作清单
- 在用户确认"已完成配置"前，不得尝试创建 PR/MR
- 后续再次触发 PR/MR 语义时，必须再次提醒初始化要求

## 技术规范

### Swagger（MUST）
- 新增接口（无论新写还是迁移）必须同步补齐 Swagger 信息（接口描述 + 关键字段描述）
- Swagger 门禁采用"告警不阻断"模式，但 Agent **必须主动关注告警**并优先补齐 Swagger 代码后再提交

### 任务级分支（MUST）
- 涉及单个系统的任意任务，必须先新切 `feature/*` 分支再开始修改
- 涉及多个系统的任务，相关系统必须使用**同名分支**并行开发
- 进入新任务不得复用旧任务分支；若已误用，需立即停止并先与用户确认处置方式

### 运营后台菜单（MUST）
- 列表类功能必须在侧边菜单中可直接展示
- "新建"类页面不作为侧边菜单独立项，统一作为对应列表页内的操作入口

---

## 飞书文档（MUST）

- 用户在本地完成文档编写后，Agent **必须询问是否上传到飞书**
- 若用户选择上传，必须继续询问上传目标空间（个人空间 / 企业空间）
- 用户选择"企业空间"时，默认使用 `space_id=7627391105121897697`（康豆产品研发）

---

## 高保真交付（MUST）

- **禁止**将 `alert`、纯文案提示或"点击后无后续承接"作为主流程最终交付行为
- `alert` 仅允许调试用途，不得作为功能闭环
- 涉及用户端闭环页面时（如会场/详情/下单/订单），提交前必须至少走通 1 条完整主链路
- 若验收发现"仅弹窗提示、无状态回流、无后续承接"的占位行为，Agent **不得宣称"已完成"**

---

## 一票否决条款（MUST）

若验收发现以下行为，Agent 不得宣称"已完成"，必须继续修复直到闭环可用：
- 仅弹窗提示、无状态回流
- 无后续承接的占位行为
- 主链路无法进入下一步页面或结果区
