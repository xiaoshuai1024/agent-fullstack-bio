# 项目脚本使用说明（scripts/**）

本文档汇总仓库内 `scripts/**` 常用脚本的用途、依赖、使用方式与常见问题，便于开发同学与 Agent 快速上手。

> 约定：除非特别说明，所有命令均在**仓库根目录**执行。

---

## 1. scripts/memory（本地知识库：mcp-memory-service）

用途：在本机启动/关闭/自检 `mcp-memory-service`（SQLite 落盘、可被 Cursor/OpenCode 通过 MCP 访问）。

### 1.1 前置依赖

- **必须**：`uvx` 可用（建议通过 `uv` 安装后确保在 PATH 中）
- **端口**：默认 `8765`
- **数据落盘**：默认写入 `scripts/memory/data/sqlite_vec.db`（该目录应保持在 `.gitignore` 中）

可选配置（本地不提交）：

- `scripts/memory/.env`（可从 `scripts/memory/.env.example` 复制后修改）

### 1.2 常用命令

- **自检并自动拉起（推荐）**

```bash
bash scripts/memory/ensure-memory.sh
```

- **前台启动（调试用）**

```bash
bash scripts/memory/up.sh
```

- **关闭服务**

```bash
bash scripts/memory/down.sh
```

### 1.3 常见问题

- **uvx 不存在**
  - 现象：`Error: uvx not found in PATH.`
  - 处理：先安装 `uv`（其内置 `uvx`），并确保 shell 的 PATH 包含安装路径

- **30s 内未就绪**
  - 现象：`mcp-memory-service did not become ready within 30s`
  - 处理：查看仓库根目录 **`logs/kangdou-memory-up.log`**（`ensure-memory.sh` 自动创建 `logs/`），多为首次依赖安装耗时或网络问题

## 1.5 scripts/git（多仓库拉取/推送）

### 1.5.1 `push-all-changes.sh`（对应 Cursor/OpenCode 命令 `/push-all`）

用途：对**有未提交变更**的 `frontend`、`backend`、`operation-backend` 与**主仓库**依次执行 `git add -A` → `git commit` → `git push -u origin HEAD`。**不创建 MR**。

```bash
bash scripts/git/push-all-changes.sh
bash scripts/git/push-all-changes.sh -m "feat: 说明"
```

- detached HEAD 的子仓库会跳过并告警。
- 默认提交说明含 UTC 日期；自定义请用 `-m`。

### 1.5.2 `pull-all-dev.sh`（对应 `/pull-all`）

```bash
bash scripts/git/pull-all-dev.sh
```

详见 `.agents/commands/pull-all.md`。

## 2. scripts/feishu（飞书 MCP 与文档上传）

用途：启动 `feishu-mcp`，以及将本地 `docs/**.md` 上传到飞书知识库（wiki）。

### 2.1 初始化本地环境文件（必做一次）

```bash
bash scripts/feishu/init-feishu-mcp.sh
```

然后编辑 `scripts/feishu/.env`，填入：

- `FEISHU_APP_ID`
- `FEISHU_APP_SECRET`

参考：`docs/dev/feishu-mcp-setup.md`

### 2.2 启动 Feishu MCP

```bash
bash scripts/feishu/start-feishu-mcp.sh
```

### 2.3 上传文档到飞书知识库

脚本：`scripts/feishu/upload-doc.sh`

必填参数：

- `--space <personal|enterprise>`
- `--file <local-file-path>`
- `--parent-node-token <wiki-node-token>`

示例（企业空间）：

```bash
bash scripts/feishu/upload-doc.sh \
  --space enterprise \
  --file docs/dev/项目脚本使用说明.md \
  --parent-node-token <your-node-token> \
  --title "项目脚本使用说明"
```

说明：

- **enterprise 默认空间**：`space_id=7627391105121897697`（康豆产品研发）
- **同名防护**：默认在创建前搜索知识库，若「同一 space + 同一父目录 + 相同标题」已存在则中止（需 `.env` 中 `FEISHU_AUTH_TYPE=user`）。确需再建副本时可加 `--allow-multiple`（不推荐）。
- **表格宽度**：上传 Markdown 表格后，脚本会用 Open API 按列设置列宽，使表格总宽度接近正文（默认 720px）；可通过环境变量 `FEISHU_TABLE_BODY_WIDTH_PX` 调整；依赖 `.env` 中的 `FEISHU_APP_ID` / `FEISHU_APP_SECRET` 换取 `tenant_access_token`。若权限失败会在 stderr 打出 Warning，表格内容仍保留。
- `--mode`：
  - `native`（默认）：Markdown → 飞书原生块（标题/列表/表格/正文样式/代码高亮/Mermaid）；**正文不出现整篇 Markdown 源码代码块或校验哈希说明**
  - `exact` / `fast`：已弃用，脚本会警告并按 `native` 处理（兼容旧命令）

## 3. scripts/codeup（云效/Codeup：MCP 启动与 MR/PR 脚本）

用途：启动云效 MCP server；在**MCP 不可用时**，提供脚本回退路径创建 MR（需显式允许）。

### 3.1 启动云效 MCP server

```bash
bash scripts/codeup/start-yunxiao-mcp.sh
```

### 3.2 云效辅助命令（组织列表 / 创建仓库）

```bash
# 列出当前 token 可见组织
bash scripts/codeup/yunxiao-mcp.sh orgs

# 创建仓库（需要 token 有权限）
bash scripts/codeup/yunxiao-mcp.sh create-repo <repo-path> [repo-name]
```

### 3.3 创建 MR（脚本回退路径）

脚本：`scripts/codeup/create-mr.sh`

说明：

- 该脚本默认遵循 **MCP-first policy**，必须显式传 `--allow-fallback` 才会走脚本回退。
- 需要 `YUNXIAO_TOKEN`（通常在 `scripts/codeup/.env` 提供）。

示例：

```bash
bash scripts/codeup/create-mr.sh \
  --source feature/xxx \
  --target dev \
  --title "chore: xxx" \
  --description "## Summary\n- ...\n" \
  --allow-fallback
```

### 3.4 一键为多仓创建 PR/MR（含子模块）

脚本：`scripts/codeup/create-pr-all.sh`

用途：当检测到子模块（如 `frontend`/`backend`）也有改动时，先分别推送并创建子模块 PR/MR，再创建根仓 PR/MR（包含 submodule pointer 更新）。

建议先 dry-run：

```bash
bash scripts/codeup/create-pr-all.sh --dry-run --allow-fallback
```
